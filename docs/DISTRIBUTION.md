# Resource Distribution Guide

Guidelines for instructors on what documentation and files to provide to different participant groups.

---

## 👨‍🏫 For the Teacher (Instructor)
Provide these to the person managing the session:
- **Guide:** `docs/teacher_guide.md`
- **PIN:** The `VITE_TEACHER_PIN` set during deployment.
- **URL:** The public URL of the application.

## 👥 For Participants (UI Users)
Provide these to students who will use the manual click interface:
- **Guide:** `docs/participant_guide.md`
- **URL:** The public URL of the application.
- **Session Code:** Generated by the teacher at the start of the session.

## 🤖 For Student Bot Developers
Provide these to students who will write optimization algorithms:
- **Guide:** `docs/student_bot_guide.md`
- **Library File:** `bot/blackbox_client.py` (The API client)
- **Template File:** `bot/student_bot_template.py` (The starter code)
- **API URL:** The backend URL (usually `http://SERVER-IP:8000`).
- **Session Code:** Provided by the teacher.

---

## 📦 Distribution Checklist

### Before the Session Starts:
1. [ ] **Verify Production URL:** Test accessibility from an external network.
2. [ ] **Prepare Bot Files:** Copy `blackbox_client.py` and `student_bot_template.py` into a `.zip` file for easier distribution to bot developers.
3. [ ] **Share Guides:** Send the appropriate `.md` files (or PDFs) to the participants.
4. [ ] **Establish PIN:** Securely communicate the Teacher PIN to instructors.

### During the Session:
1. [ ] **Announce Session Code:** Display the QR code/session code to all participants.
2. [ ] **Monitor Progress:** Ensure both manual participants and bots are appearing in the leaderboard.

### After the Session:
1. [ ] **Reveal Results:** Use the Teacher panel to reveal the function landscape.
2. [ ] **Export Data:** Download the session data for post-session analysis.
